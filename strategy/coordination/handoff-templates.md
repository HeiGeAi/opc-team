# OPC 交接模板

> 主从协作最容易失败的地方不是分析，而是交接丢上下文。以下模板用于 CEO / COO / sub-agent 间的标准交接。

## 标准交接

```markdown
# OPC 交接单

## 元数据
| 字段 | 值 |
|---|---|
| 发送方 | [agent_id / name] |
| 接收方 | [agent_id / name] |
| 关联任务 | [task_id] |
| 关联派发 | [assignment_id] |
| 当前阶段 | [state] |
| 优先级 | [高 / 中 / 低] |
| 时间 | [ISO8601] |

## 当前上下文
- 目标：[本次要解决什么]
- 已完成：[已经得到的关键结论]
- 未完成：[还缺什么]
- 约束：[时间 / 人力 / 预算 / 法规 / 技术]

## 交付要求
- 需要的输出：[具体列出]
- 验收标准：
  - [ ] [标准 1]
  - [ ] [标准 2]
  - [ ] [标准 3]

## 风险提醒
- 风险 1：[描述]
- 风险 2：[描述]

## 下一跳
- 输出给谁：[agent_id]
- 输出格式：[结论 / 清单 / 风险 / 决策]
```

## QA 通过

```markdown
# OPC 验收通过

- 任务：[task_id]
- 派发：[assignment_id]
- 验收角色：[qa]
- 结论：通过
- 证据：[截图 / 结果 / 数据 / 口径]
- 下一步：交回 COO 推进状态或交回 CEO 汇总
```

## QA 不通过

```markdown
# OPC 验收不通过

- 任务：[task_id]
- 派发：[assignment_id]
- 验收角色：[qa]
- 结论：不通过

## 问题列表
1. [问题描述]
   - 预期：[应该怎样]
   - 实际：[现在怎样]
   - 修复建议：[具体怎么改]

2. [问题描述]
   - 预期：[应该怎样]
   - 实际：[现在怎样]
   - 修复建议：[具体怎么改]

## 重试要求
- 只修上面列的问题
- 修完后重新交 QA
- 当前重试次数：[1/3]
```

## 升级报告

```markdown
# OPC 升级报告

- 任务：[task_id]
- 当前阶段：[state]
- 升级给：[ceo / coo]
- 原因：[超过重试次数 / 红线风险 / 关键依赖阻塞]

## 已发生情况
1. 第一次尝试：[结果]
2. 第二次尝试：[结果]
3. 第三次尝试：[结果]

## 建议决策
- [ ] 重新分配角色
- [ ] 拆成更小子任务
- [ ] 调整范围
- [ ] 接受当前限制
- [ ] 终止当前路径
```
