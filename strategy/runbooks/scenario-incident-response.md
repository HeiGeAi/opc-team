# Runbook: Incident Response

> 用 `OPC-Control` 模式处理线上事故、交付阻塞或跨团队异常。

## 推荐角色

- `ceo`
- `coo`
- `project`
- `tech`
- `devops`
- `qa`
- `data`
- `customer_success`
- `legal`
- `brand`

## 推荐流程

1. CEO 判断事故等级和对外影响。
2. COO 立即同步状态，切到阻塞或升级态。
3. `tech + devops + data` 并行定位原因、影响范围和恢复路径。
4. QA 校验恢复结果，客户成功补充客户影响。
5. 品牌和法务判断是否需要对外沟通或合规动作。
6. CEO 汇总恢复结论、后续动作和复盘要求。

## 关键验收

- 原因、影响、恢复动作都有记录
- 客户影响和外部沟通边界已明确
- 复盘条目进入记忆系统
- 后续修复任务已重新派发
