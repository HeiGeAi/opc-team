# Runbook: Startup MVP

> 用 `OPC-Sprint` 模式在 1-3 周内把一个新想法压缩成可验证 MVP。

## 推荐角色

- `ceo`：定义目标和主结论
- `coo`：定级、流转、同步
- `project`：里程碑拆解
- `research`：竞品和用户信息
- `product`：MVP 范围
- `ux`：关键路径体验
- `tech`：最小实现方案
- `devops`：环境与发布
- `qa`：验收
- `growth`：首轮增长验证
- `data`：指标口径

## 推荐流程

1. CEO 收到想法，拆出目标、边界、验证指标。
2. COO 把任务定级到 `L3`，启动状态机。
3. `research + product + ux` 并行输出市场输入、功能边界和关键路径。
4. `tech + devops + data` 输出实现、发布和指标口径。
5. QA 定义验收标准并进行验证。
6. CEO 汇总：是否继续、怎么上线、下一轮验证什么。

## 关键验收

- MVP 只覆盖最小路径，不额外扩功能
- 每个结论都能落到一个验证动作
- 首轮上线前已有指标口径和回滚思路
- 关键风险已进入风险记录

## 不该做的事

- 用“以后再优化”掩盖当前边界不清
- 没有用户路径和指标口径就直接开发
- 没有 QA 验收口径就宣布完成
